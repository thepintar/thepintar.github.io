Enumerable Methods

Count 'em Up

Here we are with another blog post to help illustrate some points of learning from the previous week! This week, we spent a good deal of time exploring and learning new methods to help us with some fun challenges! Sometimes it seems like the amount of methods out there are endless: the more you search, the more you find! It's comforting to know that so many neat tools exist for whatever problem you may come across in your coding experiences.

This week we'll be chatting about an Enumerable Method that I did not use in any of my code, but think could be useful in the future: "group_by". What group_by does is take a collection of objects, run them through a specified test, and groups them in a hash by their output. I'll start by sharing the example that Ruby Docs has and explaining it a little more than they do:


1   (1...6).group_by { |i| i%3 }
2   #=> {0=>[3, 6], 1=>[1, 4], 2=>[2, 5]

So check that out. By running the numbers one through six through group_by using the block "i%3", Ruby will run 1 mod 3, 2 mod 3, 3 mod 3, etc...and every object that returns the same answer will go into the same new hash, sorted by answer. As you can see in the output, 0 is returned by 3 and 6, 1 by 1 and 4 and 2 by 2 and 5!

Maybe thats still a little bit confusing to you? Well lets try a different example. Say you have a group of friends and for whatever reason you've decided that people with longer names are a little bit cooler than people with shorter names. However, you don't feel like counting up the letters in everyone's name. You could use group_by here like so:


1   ["Sherry","Terry","Hank","Georgio","Pepe","Philadelphia","Joe","Marideth"].group_by {|i| i.length}

which will return:


=> {6=>["Sherry"], 5=>["Terry"], 4=>["Hank", "Pepe"], 7=>["Georgio"], 12=>["Philadelphia"], 3=>["Joe"], 8=>["Marideth"]}

This example might be a little bit strange since when would you want to do something like that? But the method itself brings up interesting possibilities. Now we've got a hash of name lengths, and if you were to store it, you could call up just the names of whichever length you wanted.

One more example would be using it to separate a group of numbers into odds and evens. This should be pretty familiar to you now:


1   (0...10).group_by {|i| i.even?}

This will of course return a hash with two key/value pairs. The key will be "true" and its value will be an array of the even numbers and the second will be "false" with an array of the odd numbers, like so:


{true=>[0, 2, 4, 6, 8], false=>[1, 3, 5, 7, 9]}

So there you go! Hopefully you're already seeing the power of group_by, and other different ways you could use it in conjunction with other methods to get some really great results!
