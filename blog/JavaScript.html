Loop the Loop

Week 7 and we're just plugging along like a steam engine. Or like a rollercoaster car! Eh? EH?! Well listen, I'm going to go ahead and force the metaphor because its working for this entry about loops. Don't try and hold me back. Here we go.

We've learned about several methods to store a lot of data in a small place: Arrays, Hashes, and now in Javascript, objects. Its convenient to have these groupings, but only if you have a good method for retrieving the data again. Each of these methods allows for quick access of specific elements, but what if we need to access or adjust every element within an Array or Object? Now we're looking at a good time to loop it up.

Lets talk about what loops actually are. A loop is an intended repetition. Invoking a loop can be an amazing way to DRY up code, or, again, to access different values in an Array. Looking at loops and how they operate in both Ruby and Javascript offers a great example of the differences in syntax of both languages.

First lets look at some of the similarities in looping between the two. Take the "While" statement in Ruby for example.


1   while conditional [do]
2     code
3   end

This loop lets you set a condition (i.e. variable = true or counter < 10) which the loop will check the status of before every time it repeats. This loop in particular is highly readable in Ruby syntax. Another similar loop uses the "Until" statement, which looks similar.


1   until conditional [do]
2     code
3   end

Javascript has its own "While" loop, which isn't dissimilar:


1   while (conditional) {
2     code
3   }

If you choose to use curly brace formatting in Ruby, its practically identical! But lets go a little bit further and see how the similarities break down if we're using loops to iterate through an array. Here's an example in Ruby:


1   array = ["Orange","Brown","Purple","Gray"]
2   array.Each do |color|
3     puts "#{color} is my favorite color!"
4   end
5   ==>Orange is my favorite color!
6   ==>Brown is my favorite color!
7   ==>Purple is my favorite color!
8   ==>Gray is my favorite color!

As before, the Ruby syntax keeps everything really readable. Its not quite as nice looking in Javascript, where we'll use the powerful "For" loop, using the following syntax:


1   for (initializing statement; conditional statement; iterating change) {
2     code goes here;
3   }

If we want to use the same array and return the same results it'll look something like this:


1   array = ["Orange","Brown","Purple","Gray"]
2   for (var i = 0; i < array.length; i++) {
3     console.log(array[i]+"is my favorite color!");
4   }

What this loop is doing is creating a new variable "i" which it sets to 0. As long as i is less than the number of elements in the array, we'll add one to i and log each array item to the console along with the phrase "is my favorite color!". This version requires a little more thought about how you want to go about iterating across the array, but because of that its more powerful. Its similar to using the native OS on a device versus using the shell to navigate. Once you understand how to use it you have more options!

I think the lesson here is that Javascript initially looks more foreboding, especially coming from Ruby syntax, but once you get comfortable with it you start to see new possibilities.
