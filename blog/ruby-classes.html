Ruby Classes

Have a Little Class People

Welcome back! This week we're discussing ruby Classes! You can think of a class as a way to group a bunch of things that all have the same properties and rules associated with them. For instance, imagine you're a librarian who needs to keep track of a bunch of books. You could create a class called Book that would help you with this. Here's how you would start out:


1   Class Book
2   end

Pretty good, right? Well thats only the beginning. If we want our Book class to actually contain any properties or methods, we'll need to add those within. The first important step is to define the initialize method. This is what the Book class will run when you first create a new instance of Book. You'll see it below:


1   Class Book
2   def initialize(name, pagecount, isbn, checkedout)
3     @name = name
4     @pagecount = pagecount
5     @isbn = isbn
6     @checkedout = checkedout
7   end
8   end

You're probably seeing a few things you know about as well as a few new things. Lets start with line two. You'll see we're defining the method initialize to expect four arguments passed in. This means when we create a Book.new, the class will expect all of this data to be passed to it. Then, within the method you'll see those arguments being set equal to themselves with ampersands in front of them. There is something important happening here. The variables with ampersands preceding them are what we call "instance variables". They will be accessible from any method within this class, and so what we're essentially doing here is telling the class to store all four pieces of information for use with this new object.

Okay great. So now lets make a new method to make use of these instance variables.


1   Class Book
2   def initialize(name, pagecount, isbn, checkedout)
3     @name = name
4     @pagecount = pagecount
5     @isbn = isbn
6     @checkedout = checkedout
7   end
8  
9   def info
10     puts "The Title is #{@name}."
11     puts "The page count is #{@pagecount} pages."
12     puts "The ISBN is #{@isbn}."
13     @checkedout = true ? puts "It is currently checked out." : "It is currently on the shelves."
14   end
15   end

Now, when we create a new instance of Book and call info on it, it will print all the information we passed in to screen. So for instance:


1   huckfinn = Book.new("The Adventures of Huckleberry Finn", 366, 9780671420727, true
2   huckfinn.info
3   # => The Title is The Adventures of Huckleberry Finn
4   # => The page count is 366 pages.
5   # => The ISBN is 9780671420727.
6   # => It is currently checked out.

This is a pretty redimentary way to use a class, but it gets the point across. You could also attribute reader and attribute writer (or use both in attribute accessor) to make it simple to call up each piece of info or modify it without creating new methods. Next time, on Dragon Ball Z!
